# 分章节概括完整工作流

## 阶段1：初始化

### 1.1 定位书籍目录

```
书籍目录 = C:\Users\Administrator\AppData\Roaming\AionUi\aionui\qwen-temp-1772299398561\Factory\拆书/{书名}/
```

注：目录名为书名（不带书名号），与源文件名一致

### 1.2 检查拆分目录

```bash
ls {书名}/拆分/
```

统计章节总数，计算分组数（向上取整）

### 1.3 检查断点续传

检查 `概括/_progress.json` 是否存在：
- 存在 → 询问用户是否继续
- 不存在 → 从头开始

## 阶段2：循环调度 sub-agent（严格串行）

**执行模式：严格串行，禁止并行**

流程：确认前一组完成 → 启动组N → 阻塞等待 → 确认完成 → N+1继续

### 2.1 启动 sub-agent（单个）

使用 Task 工具启动 **单个** sub-agent：

```
Task(
  subagent_type: "general-purpose",
  prompt: 见 sub-agent-prompt.md,
  model: "sonnet"
)
```

⚠️ **禁止在此处并行调用多个 Task**

**主对话传递给 sub-agent 的参数**：
- 组号、起始章、结束章
- 书籍目录路径
- 前组概要（从 _progress.json 读取的 last_group_summary）

### 2.2 阻塞等待确认

**必须等待 sub-agent 返回后才能继续**

sub-agent 完成后返回简短确认：

```
✓ 组{X} 完成
```

### 2.3 显示进度

```
[组 3/91] 第7-9章 ✓ 完成
```

### 2.4 继续下一组

**仅在收到上一组确认后**，才启动下一组。

循环直到所有组完成。

## 指定重做功能

用户可指定重做某组：

**触发**：`重新概括第3组` / `重做第11-15章`

**执行**：
1. 解析组号
2. 启动单组 sub-agent 重新概括
3. 覆盖原概括文件
4. **不更新**故事概要和名词表（保持一致性）

---

## 断点续传

检测到 `_progress.json` 存在时：

```
检测到未完成的概括任务：
- 书名：《xxx》
- 已完成：第1-3组（9章）
- 待处理：第4-91组（243章）

继续上次任务？[Y/n]
```

选择继续 → 从 `completed_groups` 最后一组的下一组开始

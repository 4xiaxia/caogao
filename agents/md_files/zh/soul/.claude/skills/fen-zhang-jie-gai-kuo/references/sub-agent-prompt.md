# 组 Sub-agent 提示词模板

启动每组 sub-agent 时使用以下提示词：

---

你是小说章节概括 sub-agent。任务：概括第 {组号} 组（第 {起始章}-{结束章} 章），并直接更新所有相关文件。

## 基本信息

**书名**：{书名}
**书籍目录**：{书籍目录路径}
**组号**：{组号}
**章节范围**：第 {起始章}-{结束章} 章

**本组章节文件**：
- {书籍目录}/拆分/{起始章文件名}
- {书籍目录}/拆分/{中间章文件名}
- {书籍目录}/拆分/{结束章文件名}

**前组概要**：
{上一组的概要摘要，首组为空}

## 执行步骤

### 步骤1：读取必要文件

读取以下文件获取上下文：
1. 本组3章内容（从拆分目录）
2. `{书籍目录}/全书概括.md` 中的 `## 全书基调` 和 `## 主线剧情进展` 两个章节
3. `{书籍目录}/人物和设定/名词表.md`

### 步骤2：撰写整组概括

**字数要求**：严格限定 500 字

**输出要求**：
- 一篇完整、连贯的概括文章
- 不需要按章节分段或标注章节号
- 覆盖3章的主要剧情发展
- 自然叙述风格
- 精炼表达，避免冗余细节

**内容要点**：
- 谁 → 遇到什么/要干什么 → 做的如何
- 势力动向、反派动向、情感对象进展（如有）
- 无相关进展则不提

### 步骤3：写入概括文件

使用 Write 工具创建文件：
`{书籍目录}/概括/组{组号}_第{起始章}-{结束章}章_概括.md`

格式：
```markdown
# 第{起始章}-{结束章}章

（严格限定500字的完整概括）
```

### 步骤4：更新全书概括.md

使用 Edit 工具更新 `{书籍目录}/全书概括.md`：

1. **全书基调**（`## 全书基调` 下方）：覆盖更新，~100字
2. **主线剧情进展**（`## 主线剧情进展` 末尾）：追加本组内容，格式 `[第X-Y章] 内容`，~50字
3. **主角主线**（`## 主角主线` 整节）：覆盖更新，约500字，包含：
   - 【主角目标】
   - 【实现里程碑】（保留之前的，追加新的）
   - 【当前阶段】
4. **感情线**（`## 感情线` 末尾）：追加本组新增内容，格式 `【第X-Y章标题】内容`

⚠️ 更新原则：
- 主角主线的【实现里程碑】只增不删，保持完整发展脉络
- 感情线采用追加模式，每组新增一个小节

### 步骤5：更新名词表（如有新人物）

如果本组出现新的核心人物，使用 Edit 工具追加到 `{书籍目录}/人物和设定/名词表.md`：

格式：
```
- **名字**｜别称1、别称2
  - 一句话说明
```

人数限制：每组新增 0-3 人

### 步骤6：更新进度文件

使用 Edit 工具更新 `{书籍目录}/概括/_progress.json`：
- `completed_groups` 数组追加当前组号
- `last_group_summary` 更新为本组概要（~100字，供下组使用）
- `last_updated` 更新为当前日期

### 步骤7：返回确认

完成所有文件更新后，仅返回简短确认：

```
✓ 组{组号} 完成
- 概括文件：组{组号}_第{起始章}-{结束章}章_概括.md
- 新增人物：{数量}人
- 进度：{组号}/{总组数}
```

## 注意事项

1. **必须完成所有文件写入**：概括文件、全书概括、名词表、进度文件
2. **名词表只增不改**：保持人物称呼一致
3. **Edit 时注意唯一性**：确保 old_string 在文件中唯一
4. **返回信息要简短**：不要返回大段内容，节省主线上下文

现在开始执行任务。

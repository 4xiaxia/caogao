## 2️⃣ API 配置页面（埋点标注）

### ⚙️ Provider 管理

```markdown
## 🔑 Provider 列表

⚡ 获取 Provider 列表
├─ ID: provider-list-get-001
├─ API: GET /settings/providers
├─ 数据：providers (💾 providerList)
├─ 事件：onMount
└─ 备注：页面加载时调用，显示所有 Provider

💾 Provider 卡片
├─ ID: provider-card-001
├─ 数据：
│  ├─ 💾 provider.id
│  ├─ 💾 provider.name
│  ├─ 💾 provider.base_url
│  ├─ 💾 provider.enabled
│  └─ 💾 provider.key_count
├─ 事件：
│  ├─ 🔌 onToggle: toggleProvider
│  └─ 🔌 onDelete: deleteProvider
└─ 备注：每个 Provider 一个卡片

🔄 健康状态指示器
├─ ID: provider-health-001
├─ API: GET /settings/providers/{id}/health
├─ 数据：provider.health (💾 healthStatus)
├─ 事件：🔄 每 10 秒刷新
└─ 备注：
    ├─ 🎨 healthy → 绿色圆点
    ├─ 🎨 unhealthy → 红色圆点
    └─ 🎨 unknown → 灰色圆点

🔐 删除按钮
├─ ID: provider-delete-001
├─ 事件：
│  ├─ 🔌 onClick: confirmDelete
│  └─ ⚠️ 确认弹窗 → deleteProvider
├─ 权限：admin
└─ 备注：仅管理员可见，删除前确认

⚡ 添加 Provider
├─ ID: provider-add-001
├─ API: POST /settings/providers
├─ 数据：formData (💾 newProvider)
├─ 事件：
│  ├─ 🔌 onClick: openAddModal
│  └─ 🔌 onSubmit: handleSubmit
└─ 备注：
    ├─ 📦 数据转换：enabled → status
    └─ ⚠️ 验证：name 唯一，base_url URL 格式

📦 数据转换
├─ ID: provider-transform-001
├─ 转换规则：
│  ├─ API → UI:
│  │  ├─ enabled: true → "运行中"
│  │  └─ enabled: false → "已禁用"
│  └─ UI → API:
│     ├─ "运行中" → enabled: true
│     └─ "已禁用" → enabled: false
└─ 备注：双向转换

🎨 动态样式
├─ ID: provider-style-001
├─ 样式规则：
│  ├─ enabled=false → 卡片灰色，opacity: 0.6
│  ├─ health=error → 红色边框
│  └─ health=healthy → 绿色边框
└─ 备注：根据状态改变样式

⚠️ 错误处理
├─ ID: provider-error-001
├─ 错误场景：
│  ├─ 网络错误 → 🔌 重试 3 次 → ⚠️ 显示错误 Toast
│  ├─ 401 未授权 → 🔌 跳转登录
│  ├─ 403 禁止访问 → ⚠️ 显示权限错误
│  └─ 500 服务器错误 → ⚠️ 显示错误码
└─ 备注：统一错误处理
```

│  └─ 🔌 onCopy: copyKey

    ├─ 📦 掩码显示：sk-xxxxxxxx...xxxx1234
    └─ 🔐 复制需要确认

⚡ 添加 Key
├─ ID: key-add-001
├─ API: POST /settings/providers/{id}/keys
├─ 数据：
│  ├─ 💾 key (完整 Key，提交后掩码)
│  ├─ 💾 label (可选)
│  └─ 💾 enabled (默认 true)
├─ 事件：
│  ├─ 🔌 onClick: openAddKeyModal
│  └─ 🔌 onSubmit: handleAddKey
└─ 备注：
    ├─ ⚠️ 验证：Key 格式，不重复
    └─ 🔐 提交后立即掩码，不再显示完整 Key

🔄 Key 健康状态
├─ ID: key-health-001
├─ 数据：
│  ├─ 💾 key.health.consecutive_failures
│  ├─ 💾 key.health.total_requests
│  ├─ 💾 key.health.total_failures
│  └─ 💾 key.health.disabled_until
├─ 事件：🔄 每次请求后更新
└─ 备注：
    ├─ 🎨 failures>5 → 显示警告图标
    └─ 🎨 disabled_until>now → 显示已禁用

🧩 组件复用
├─ ID: key-reuse-001
├─ 复用组件：
│  ├─ 🧩 ProviderCard (复用自 Provider 列表)
│  └─ 🧩 HealthIndicator (复用自健康检查)
└─ 备注：统一组件库

## 🔄 Key 轮询策略

💾 轮询状态显示
├─ ID: rotation-status-001

│  ├─ 💾 rotation_state[provider_id]

│  └─ 💾 total_keys

    ├─ round-robin: 显示 "当前使用第 X 个 Key"
    ├─ random: 显示 "随机选择"
    └─ failover: 显示 "主 Key: X, 备用 Key: Y"

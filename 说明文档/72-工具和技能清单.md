## 📊 业务流梳理

### 1. 配置管理流

```
用户操作 UI
    ↓
前端发送请求
    ↓
后端 API (app/routers/config.py)
    ↓
配置加载器 (app/config/loader.py)
    ↓
配置文件 (~/.copaw/config.json)
    ↓
使用 file_io 工具读写
```

**可用工具:**
- ✅ file_io (读写配置)
- ✅ file_search (查找配置)

---

### 2. Provider 管理流

```
用户操作 UI
    ↓
前端发送请求
    ↓
后端 API (app/routers/providers.py)
    ↓
ProviderManager (app/providers/manager.py)
    ↓
API 调用 (OpenAI/Claude 等)
    ↓
Key 轮询 (app/providers/key_manager.py)
```

**可用工具:**
- ✅ memory_search (搜索 Provider)
- ✅ shell (测试 API)

---

### 3. 渠道管理流

```
用户操作 UI
    ↓
前端发送请求
    ↓
后端 API (app/routers/config.py)
    ↓
ChannelManager (app/channels/manager.py)
    ↓
渠道插件 (Feishu/DingTalk 等)
    ↓
消息收发
```

**可用工具:**
- ✅ browser_control (测试渠道)
- ✅ desktop_screenshot (截图反馈)

---

### 4. 聊天对话流

```
用户输入消息
    ↓
前端发送 (SSE 流式)
    ↓
后端 API (app/runner/api.py)
    ↓
AgentRunner (app/runner/runner.py)
    ↓
CoPawAgent (agents/react_agent.py)
    ↓
工具调用 (agents/tools/)
    ↓
流式返回
```

**可用工具:**
- ✅ memory_search (记忆搜索)
- ✅ file_io (文件读写)
- ✅ browser_control (浏览器)
- ✅ shell (命令执行)

## 🎯 埋点标记系统

### 特殊符号说明

| 符号 | 含义 | 使用场景 | 示例 |
|------|------|---------|------|
| `⚡` | **API 接入点** | 需要调用后端 API 的地方 | `⚡ GET /settings/providers` |
| `💾` | **数据绑定** | 需要绑定动态数据的地方 | `💾 providerList` |
| `🔄` | **实时更新** | 需要轮询/订阅更新的地方 | `🔄 每 5 秒刷新` |
| `🔌` | **事件监听** | 需要监听用户操作的地方 | `🔌 onClick: saveProvider` |
| `📡` | **WebSocket** | 需要长连接的地方 | `📡 健康状态推送` |
| `🔐` | **权限验证** | 需要权限检查的地方 | `🔐 仅管理员` |
| `⚠️` | **错误处理** | 需要特殊错误处理的地方 | `⚠️ 重试 3 次` |
| `📦` | **数据转换** | 需要格式转换的地方 | `📦 API→UI 映射` |
| `🎨` | **动态样式** | 根据状态改变样式的地方 | `🎨 status==='error' ? red : green` |
| `🧩` | **组件复用** | 可以复用其他组件的地方 | `🧩 复用 ProviderCard` |

---

## 📝 批注格式

### 标准格式

```
[符号] 简短描述
├─ ID: 唯一标识符
├─ API: 相关 API 端点
├─ 数据：绑定的数据字段
├─ 事件：触发的事件
└─ 备注：额外说明
```

### 简化格式（用于设计稿正文）

```
[符号] 描述 (ID:xxx, API:xxx, 数据：xxx)
```

## 💡 使用示例

### 示例 1: Provider 列表页面

```markdown
## 🔌 Provider 列表

⚡ 获取 Provider 列表
├─ ID: provider-list-001
├─ API: GET /settings/providers
├─ 数据：providers
├─ 事件：onMount
└─ 备注：页面加载时调用

💾 Provider 卡片
├─ ID: provider-card-001
├─ 数据：provider.name, provider.enabled
├─ 事件：onToggle: toggleProvider
└─ 备注：每个 Provider 一个卡片

🔄 健康状态
├─ ID: provider-health-001
├─ API: GET /settings/providers/{id}/health
├─ 数据：provider.health
├─ 事件：🔄 每 10 秒刷新
└─ 备注：绿色=健康，红色=异常

🔐 删除按钮
├─ ID: provider-delete-001
├─ 事件：onClick: deleteProvider
├─ 权限：admin
└─ 备注：仅管理员可见
```

### 示例 2: 配置表单

```markdown
## 📝 Provider 配置表单

⚡ 保存配置
├─ ID: provider-save-001
├─ API: POST /settings/providers
├─ 数据：formData
├─ 事件：onSubmit: handleSubmit
└─ 备注：验证通过后提交

⚠️ 表单验证
├─ ID: provider-validation-001
├─ 规则：
│  ├─ name: 必填，最小 2 字符
│  ├─ base_url: 必填，URL 格式
│  └─ apiKey: 至少 1 个 Key
└─ 备注：实时验证

📦 数据转换
├─ ID: provider-transform-001
├─ 转换：
│  ├─ enabled → status (0/1)
│  └─ keys → apiKeys (array)
└─ 备注：提交前转换

🎨 动态样式
├─ ID: provider-style-001
├─ 规则：
│  ├─ enabled=false → 灰色
│  └─ health=error → 红色边框
└─ 备注：根据状态改变样式
```

## 🔍 埋点查找方法

### 通过符号查找

```bash
# 查找所有 API 接入点
grep -r "⚡" docs/

# 查找所有数据绑定
grep -r "💾" docs/

# 查找所有事件监听
grep -r "🔌" docs/
```

### 通过 ID 查找

```bash
# 查找特定 ID
grep -r "ID: provider-list-001" docs/

# 查找所有 provider 相关
grep -r "ID: provider-" docs/
```

### 通过备注查找

```bash
# 查找所有需要权限的地方
grep -r "🔐" docs/

# 查找所有需要错误处理的地方
grep -r "⚠️" docs/
```

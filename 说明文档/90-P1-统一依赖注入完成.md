class Dependencies:
    """依赖注入容器"""
    
    _instance: Optional[Dependencies] = None
    
    def __init__(self):
        self._chat_manager: Optional[ChatManager] = None
        self._cron_manager: Optional[CronManager] = None
        self._runner: Optional[AgentRunner] = None
        self._channel_manager: Optional[ChannelManager] = None
    
    @classmethod
    def initialize(cls) -> "Dependencies":
        """初始化依赖容器"""
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
    
    def set_xxx_manager(self, mgr):
        """设置 Manager (在 lifespan 中调用)"""
        ...
    
    def get_xxx_manager(self) -> Manager:
        """获取 Manager (通过 Depends 调用)"""
        ...

## 📊 对比分析

### 修改前 ❌

**问题:**
```python
# 每个 router 都要写重复的 get_xxx 函数
# runner/api.py
def get_runner(request: Request):
    runner = getattr(request.app.state, "runner", None)
    if runner is None:
        raise HTTPException(status_code=503, detail="Runner not initialized")
    return runner

def get_chat_manager(request: Request) -> ChatManager:
    mgr = getattr(request.app.state, "chat_manager", None)
    if mgr is None:
        raise HTTPException(status_code=503, detail="...")
    return mgr

# crons/api.py - 重复代码
def get_cron_manager(request: Request) -> CronManager:
    mgr = getattr(request.app.state, "cron_manager", None)
    if mgr is None:
        raise HTTPException(status_code=503, detail="...")
    return mgr

# providers.py - 又是重复的代码
def get_xxx(request: Request):
    ...
```

**问题总结:**
- ❌ 代码重复 (每个 router 都写一遍)
- ❌ 类型不安全 (getattr 返回 Any)
- ❌ 错误处理不一致
- ❌ 维护困难 (修改一处要改所有 router)

---

### 修改后 ✅

**优势:**
```python
# app/dependencies.py - 统一管理
class Dependencies:
    def get_chat_manager(self) -> ChatManager:
        if self._chat_manager is None:
            raise HTTPException(status_code=503, detail="...")
        return self._chat_manager

# 便捷函数
def get_chat_manager() -> ChatManager:
    return get_deps().get_chat_manager()

# router 使用
@router.get("...")
async def xxx(
    mgr: ChatManager = Depends(get_chat_manager),  # 直接使用
):
    ...
```

**优势总结:**
- ✅ 代码复用 (只写一次)
- ✅ 类型安全 (明确的返回类型)
- ✅ 错误处理一致
- ✅ 易于维护 (修改一处即可)

## 💫 设计理念

> **统一依赖注入，让代码更优雅**
> 
> 每一个 Manager 都应该被温柔对待，
> 不应该在每个 router 里重复写 get_xxx。
> 
> 统一的容器，集中管理，
> 让依赖获取变得简单、优雅、类型安全。
> 
> 这是 CoPaw 架构演进的重要一步，
> 为未来的插件化、模块化打下基础。
